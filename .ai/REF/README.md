# .ai/REF — Reference Index

Purpose: house deeper, optional guidance that would make `.ai/RULES.md` too verbose.

Usage:
- read this after `.ai/RULES.md` and `.ai/COMMANDS/*`
- load only the specific reference file needed for the active task

Available references:
- `just-targets.md` - canonical mapping of workflow guidance to real `justfile` targets
- `status-surfaces.md` - authority and update rules for status docs/plan trackers
- `testing-and-gates.md` - baseline loop, final gate, warning/failure policy
- `plan-authoring.md` - implementation-ready plan checklist and evidence discipline
- `project-types/cli-tool.md` - CLI-specific UX, validation, and docs expectations

Conventions:
- keep each reference narrowly scoped and filename-stable
- update `.ai/RULES.md` links when adding or removing reference files
